## 🟠 Crashes & Stability

### 1. move_prim/MovePrims Crash on Extension Startup — Segfault (5.0.0)
- **URL:** https://github.com/isaac-sim/IsaacSim/issues/556
- **Version:** Isaac Sim 5.0.0
- **Category:** crashes-stability
- **Severity:** pain
- **Source:** GitHub | 2026-03-30
- **Summary:** Calling move_prim() or MovePrims during extension startup causes segfault. Works fine after initialization. Root cause: MovePrimCommand re-fetches USD stage without null-checking — if stage isn't ready, C++ bindings crash instead of raising Python exception. Fix: defer to StageEventType.ASSETS_LOADED event. 3 comments, still OPEN.
- **Signal:** Extension lifecycle timing bug. Affects anyone building custom extensions that manipulate prims on startup. Well-diagnosed by reporter with clear fix path.

  - **Isaac Sim & MuJoCo handle physics well but don't solve environment generation** — users still manually build scenes

- **Signal:** Confirms our tracked pain points: asset pipeline gap, environment generation unsolved, DX fragmentation. The "prompt → simulation" use case is emerging but blocked by fundamental architecture gaps.
