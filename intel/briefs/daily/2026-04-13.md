### 1. Isaac Sim Standalone Installer Missing USD Development Headers (5.1.0)
- **URL:** https://github.com/isaac-sim/IsaacSim/issues/560
- **Version:** Isaac Sim 5.1.0-rc.19 (OpenUSD 24.05)
- **Category:** tooling-dx
- **Severity:** pain
- **Source:** GitHub | 2026-04-01
- **Summary:** Standalone installer lacks USD C++ development headers (pxr/) and pxrConfig.cmake. Impossible to compile custom USD plugins (ArResolver, custom schemas) against the installation. Users must either build from source or manually patch headers for ABI namespace. 2 comments, still OPEN.
- **Signal:** Blocks anyone building custom USD plugins — common need for enterprise asset pipelines. "Neither approach is practical for end users."

### 2. IsaacSim/IsaacLab Hard to Use by Multiple Users on Same PC (5.1.0 / 6.0.0)
- **URL:** https://github.com/isaac-sim/IsaacLab/issues/5158
- **Version:** Isaac Sim 5.1.0 + Isaac Lab 2.3.x AND Isaac Sim 6.0.0 + Isaac Lab 3.0.0-beta | Ubuntu 22.04
- **Category:** tooling-dx
- **Severity:** pain
- **Source:** GitHub | 2026-04-02
- **Summary:** Five distinct permission problems when multiple users share one PC: (1) extension cache symlinks to installer's home, (2) log file write restrictions, (3) datastore locking errors, (4) /tmp/IsaacLab owned by first user blocking URDF imports, (5) Python install permission denied. Affects both 5.1 and 6.0. Group permissions workaround doesn't fully resolve. 1 comment, still OPEN.
- **Signal:** Multi-user lab/team environments are common in academia and enterprise. This is a deployment-blocking DX issue for shared workstations.
